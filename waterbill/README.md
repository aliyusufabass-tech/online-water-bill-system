# Online Water Bill System

## Deployment (Vercel + Render)

### Architecture
- Frontend (React/Vite): Vercel
- Backend API (Django): Render
- Database (Postgres): Render managed database

### 1. Deploy Backend on Render
- In Render, create a new Blueprint and select this repository.
- Render will use `render.yaml` to create:
  - Web service: `online-water-bill-api`
  - Postgres database: `online-water-bill-db`

### 2. Set Backend Environment Variables in Render
- After the service is created, set/update these values in the Render web service:
- `ALLOWED_HOSTS=<your-render-service>.onrender.com`
- `CORS_ALLOWED_ORIGINS=https://<your-vercel-app>.vercel.app`
- `CSRF_TRUSTED_ORIGINS=https://<your-vercel-app>.vercel.app`
- `DEBUG=false`

Notes:
- `SECRET_KEY` is generated by `render.yaml`.
- `DATABASE_URL` is linked automatically from the Render Postgres database.

### 3. Deploy Frontend on Vercel
- Import the same GitHub repository into Vercel.
- `vercel.json` is already configured to build the nested frontend at `waterbill/frontend`.

### 4. Set Frontend Environment Variable in Vercel
- Add this variable in the Vercel project:
- `VITE_API_BASE_URL=https://<your-render-service>.onrender.com/api`

### 5. Redeploy
- Redeploy Render after backend env var changes.
- Redeploy Vercel after setting `VITE_API_BASE_URL`.

## Local Development

### Backend
```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

### Frontend
```bash
cd frontend
npm install
npm run dev
```

## Security
- Do not commit real database passwords or production secrets.
- Rotate any secrets that were previously committed to git history.
